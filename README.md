Session-Based RAG Application

Setup Instructions

This project requires two active terminal windows to run the separated frontend and backend environments.

A. Backend Setup

Navigate to the backend directory: cd backend
Install dependencies: npm install
Create a .env file in the backend folder and add your Google Gemini API key: i.PORT=3000 ii.GEMINI_API_KEY=your_actual_api_key_here
Start the development server: npm run dev
B. Frontend Setup

Navigate to the frontend directory: cd frontend
Install dependencies: npm install
Ensure your Google OAuth Client ID is set in App.tsx (for the login screen).
Start the Vite development server: npm run dev
C. Architecture Overview :

The application follows a decoupled client-server architecture:

Frontend: Built with React (Vite), TypeScript, and Tailwind CSS. It handles file validation, chunk tracking, similarity threshold adjustments, and renders the chat interface.
Backend: A Node.js/Express server that acts as the orchestration layer. It handles file parsing in memory, interacts with the Gemini API for embeddings and text generation, and manages the mathematical retrieval logic.
Data Storage: To meet strict assessment constraints, an in-memory vector store (via JavaScript Map) is utilized. Data is strictly isolated using sessionId generated by the frontend.
D. Retrieval Flow

Embedding: When a document is uploaded, chunks are concurrently embedded using Google's gemini-embedding-001 model and stored in the memory dictionary under the user's sessionId.

Querying: The user's question is embedded using the same model.
Manual Vector Math: The system calculates the Cosine Similarity mathematically between the question vector and all stored document vectors for that session.
Top-K: The array is sorted by similarity score, and the Top 3 (K=3) most relevant chunks are retrieved.
